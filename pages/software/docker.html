Introduction

Docker is a platform that enables developers to build, share, and run applications in containers. Containers are lightweight, standalone packages that include everything needed to run an application: code, runtime, system tools, libraries, and settings.

Unlike virtual machines, containers share the host system's kernel and don't require a full OS for each application. This makes them incredibly fast to start, efficient with resources, and consistent across different environments. "Works on my machine" becomes a thing of the past with Docker.

Docker has revolutionized software development and deployment, becoming an essential tool for DevOps, microservices architecture, and cloud-native applications. Whether you're developing locally, deploying to production, or managing complex infrastructure, Docker simplifies the entire process.

Key Features

  • Containerization: Package applications with all dependencies into portable containers
  • Docker Hub: Access thousands of pre-built images from the official registry
  • Docker Compose: Define and run multi-container applications with YAML files
  • Cross-Platform: Same containers run on Windows, macOS, Linux, and cloud
  • Isolation: Each container runs in its own isolated environment
  • Version Control: Track changes to containers like code with image layers
  • Networking: Built-in networking for container communication
  • Volume Management: Persist data outside containers
  • Fast Startup: Containers start in seconds, not minutes
  • Resource Efficient: Uses less memory and CPU than VMs
  • CI/CD Integration: Easily integrate with Jenkins, GitHub Actions, GitLab CI
  • Kubernetes Ready: Works seamlessly with container orchestration

Download & Install on Windows

Official Download: https://www.docker.com/products/docker-desktop/

Prerequisites:

  1. Enable WSL 2: Open PowerShell as Admin and run:
    wsl --install
  2. Restart your computer
  3. Enable virtualization in BIOS (usually under CPU settings)

Installation Steps:

  1. Download Docker Desktop for Windows from the official website
  2. Run "Docker Desktop Installer.exe"
  3. Keep "Use WSL 2 instead of Hyper-V" checked (recommended)
  4. Click OK and wait for installation
  5. Restart your computer when prompted
  6. Launch Docker Desktop from the Start menu
  7. Accept the Service Agreement
  8. Wait for Docker to start (whale icon in system tray turns steady)

Download & Install on macOS

Official Download: https://www.docker.com/products/docker-desktop/
  1. Download the appropriate version:
    • Apple Silicon (M1/M2/M3): Download "Apple Chip" version
    • Intel Mac: Download "Intel Chip" version
  2. Open the .dmg file
  3. Drag Docker.app to Applications folder
  4. Open Docker from Applications
  5. Grant permissions when prompted
  6. Enter your password to install helper components
  7. Wait for Docker to start (whale icon appears in menu bar)

Alternative: Using Homebrew

brew install --cask docker

Download & Install on Linux

Official Guide: https://docs.docker.com/engine/install/

Ubuntu/Debian

# Remove old versions sudo apt-get remove docker docker-engine docker.io containerd runc # Set up repository sudo apt-get update sudo apt-get install ca-certificates curl gnupg # Add Docker's official GPG key sudo install -m 0755 -d /etc/apt/keyrings curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg sudo chmod a+r /etc/apt/keyrings/docker.gpg # Set up repository echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null # Install Docker Engine sudo apt-get update sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin # Add your user to docker group (run without sudo) sudo usermod -aG docker $USER

Fedora

sudo dnf install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin sudo systemctl start docker sudo systemctl enable docker

How to Use (Beginner Guide)

1. Verify Installation

docker --version docker run hello-world

2. Basic Commands

CommandDescription
docker pull [image]Download an image from Docker Hub
docker run [image]Run a container from an image
docker psList running containers
docker ps -aList all containers
docker stop [id]Stop a running container
docker rm [id]Remove a container
docker imagesList downloaded images
docker rmi [image]Remove an image

3. Run Your First Container

# Run nginx web server docker run -d -p 8080:80 nginx # Open http://localhost:8080 in your browser

4. Create a Dockerfile

# Dockerfile example for a Node.js app FROM node:18 WORKDIR /app COPY package*.json ./ RUN npm install COPY . . EXPOSE 3000 CMD ["npm", "start"]

5. Build and Run Custom Image

# Build image docker build -t my-app . # Run container docker run -d -p 3000:3000 my-app

Common Problems & Fixes

Problem 1: Docker daemon not running

  • Windows/Mac: Make sure Docker Desktop is running (check system tray)
  • Linux: Run sudo systemctl start docker

Problem 2: Permission denied on Linux

sudo usermod -aG docker $USER # Then log out and log back in

Problem 3: WSL 2 not installed (Windows)

wsl --install wsl --set-default-version 2

Problem 4: Port already in use

  • Check what's using the port: netstat -ano | findstr :8080
  • Use a different port: docker run -p 8081:80 nginx

Problem 5: Out of disk space

# Remove unused containers, images, and volumes docker system prune -a
