Introduction

LAMP Stack is the original and most widely-used open-source web development platform, consisting of Linux (operating system), Apache (web server), MySQL/MariaDB (database), and PHP/Perl/Python (programming languages). This combination has powered millions of websites and web applications for over two decades.

Bitnami LAMP Stack provides a pre-configured, easy-to-install package that bundles all these components together. Whether you're setting up a development environment or deploying a production server, Bitnami simplifies the installation and configuration process significantly.

The LAMP stack remains the foundation for popular applications like WordPress, Drupal, Joomla, and countless custom PHP applications. Its reliability, performance, and extensive community support make it an excellent choice for web developers of all levels.

Key Features

  • Complete Stack: Apache, MySQL/MariaDB, PHP, and phpMyAdmin bundled together and pre-configured
  • Cross-Platform: Available for Linux, Windows, and macOS through Bitnami installers
  • Self-Contained: Runs independently without affecting system-wide configurations
  • Easy Management: Simple scripts to start, stop, and restart all services
  • Production-Ready: Optimized configurations suitable for development and production use
  • Cloud Ready: Available as cloud images for AWS, Google Cloud, Azure, and other platforms

Download & Installation

Official Download Link: https://bitnami.com/stack/lamp

Option 1: Bitnami Installer (All Platforms)

  1. Visit the Bitnami LAMP Stack page: bitnami.com/stack/lamp
  2. Select your operating system (Linux, Windows, or macOS)
  3. Download the installer package
  4. Run the installer and follow the wizard
  5. Choose installation directory
  6. Set the MySQL root password when prompted
  7. Complete installation and start services

Option 2: Native Linux Installation (Ubuntu/Debian)

# Update package list sudo apt update # Install Apache sudo apt install apache2 # Install MySQL sudo apt install mysql-server sudo mysql_secure_installation # Install PHP sudo apt install php libapache2-mod-php php-mysql # Restart Apache sudo systemctl restart apache2

Option 3: Native Linux Installation (CentOS/RHEL)

# Install Apache sudo dnf install httpd sudo systemctl start httpd sudo systemctl enable httpd # Install MySQL/MariaDB sudo dnf install mariadb-server sudo systemctl start mariadb sudo mysql_secure_installation # Install PHP sudo dnf install php php-mysqlnd sudo systemctl restart httpd

Option 4: Docker Container

# Pull and run Bitnami LAMP container docker run -d --name lamp -p 80:80 -p 443:443 bitnami/lamp # Or use Docker Compose for more control

How to Use

1. Managing Services (Bitnami Installation)

# Navigate to installation directory cd /opt/lampstack # Start all services sudo ./ctlscript.sh start # Stop all services sudo ./ctlscript.sh stop # Restart all services sudo ./ctlscript.sh restart # Check status sudo ./ctlscript.sh status

2. Document Root Locations

Installation Type Document Root
Bitnami /opt/lampstack/apache2/htdocs/
Native Ubuntu/Debian /var/www/html/
Native CentOS/RHEL /var/www/html/

3. Accessing phpMyAdmin

  • Open browser and go to: http://localhost/phpmyadmin
  • Login with MySQL credentials (default user: root)
  • Create databases, manage tables, and run queries

4. Creating a PHP Test File

Create a file named info.php in your document root:

<?php phpinfo(); ?>

Access it at http://localhost/info.php to verify PHP is working.

5. Configuring Virtual Hosts

For Apache on Linux, create a new config file in /etc/apache2/sites-available/:

<VirtualHost *:80> ServerName mysite.local DocumentRoot /var/www/mysite <Directory /var/www/mysite> AllowOverride All Require all granted </Directory> </VirtualHost>

Common Problems & Fixes

Problem 1: Apache won't start - port 80 in use

Solution:

  • Check what's using port 80: sudo lsof -i :80 or sudo netstat -tlnp | grep :80
  • Stop the conflicting service or change Apache port in httpd.conf
  • Common conflicts: nginx, other Apache instances, or Skype

Problem 2: MySQL access denied for root

Solution:

  • For Bitnami, check password in /opt/lampstack/mysql/data/bitnami_credentials
  • Reset password: sudo mysql then ALTER USER 'root'@'localhost' IDENTIFIED BY 'newpassword';
  • Update application config files with new credentials

Problem 3: PHP files download instead of executing

Solution:

  • Ensure PHP module is enabled: sudo a2enmod php8.1
  • Add PHP handler in Apache config
  • Restart Apache: sudo systemctl restart apache2

Problem 4: Permission denied errors

Solution:

  • Set proper ownership: sudo chown -R www-data:www-data /var/www/html
  • Set proper permissions: sudo chmod -R 755 /var/www/html
  • For writable directories: sudo chmod -R 775 /var/www/html/uploads

Problem 5: .htaccess not working

Solution:

  • Enable mod_rewrite: sudo a2enmod rewrite
  • Set AllowOverride All in Apache virtual host config
  • Restart Apache after changes

Problem 6: Cannot connect to MySQL from PHP

Solution:

  • Install PHP MySQL extension: sudo apt install php-mysql
  • Verify MySQL is running: sudo systemctl status mysql
  • Check connection settings in your PHP code
  • Restart Apache after installing extensions
