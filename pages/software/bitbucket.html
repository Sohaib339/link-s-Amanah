Introduction

Bitbucket is a web-based Git repository hosting service owned by Atlassian. It provides source code management, CI/CD pipelines, and collaboration tools for development teams of all sizes.

Bitbucket integrates seamlessly with other Atlassian products like Jira, Trello, and Confluence, making it an excellent choice for teams already using the Atlassian ecosystem for project management and documentation.

Key Features

  • Git Repository Hosting: Unlimited private repositories with Git LFS support
  • Bitbucket Pipelines: Built-in CI/CD for automated builds, tests, and deployments
  • Pull Requests: Code review workflow with inline comments and approval system
  • Branch Permissions: Protect branches and enforce merge requirements
  • Jira Integration: Link commits, branches, and PRs to Jira issues automatically
  • Bitbucket Deployments: Track deployment status and environments
  • Code Search: Search across repositories with advanced filtering
  • Security Scanning: Detect vulnerabilities in dependencies and code

Download & Install

Official Access: Bitbucket Cloud

Bitbucket Cloud (Web-based)

  1. Visit bitbucket.org in your web browser
  2. Click "Get started for free"
  3. Sign up with email or use Google/Microsoft account
  4. Create your first workspace
  5. Start creating repositories

Git Client Setup (Windows)

  1. Download Git from git-scm.com
  2. Install with default options
  3. Configure: git config --global user.name "Your Name"
  4. Configure: git config --global user.email "your@email.com"
  5. Generate SSH key: ssh-keygen -t ed25519
  6. Add SSH key to Bitbucket account settings

Git Client Setup (macOS/Linux)

  1. macOS: Install via Xcode command line tools or Homebrew
  2. Linux: Install via package manager (apt, yum, dnf)
  3. Configure Git username and email
  4. Generate and add SSH key to Bitbucket

How to Use (Beginner Guide)

1. Creating a Repository

  • Log in to Bitbucket and go to your workspace
  • Click "Create repository"
  • Enter repository name and description
  • Choose access level (public or private)
  • Click "Create repository"

2. Cloning a Repository

  1. Navigate to your repository on Bitbucket
  2. Click "Clone" and copy the URL
  3. Open terminal and run: git clone [URL]
  4. Navigate into the cloned directory

3. Creating a Pull Request

  • Create a new branch: git checkout -b feature-branch
  • Make changes and commit: git add . && git commit -m "message"
  • Push branch: git push origin feature-branch
  • Go to Bitbucket and click "Create pull request"
  • Add reviewers and description, then submit

Common Problems & Fixes

Problem 1: Authentication failed when pushing

  • Use SSH instead of HTTPS for authentication
  • Create an App Password in Bitbucket settings
  • Update remote URL: git remote set-url origin [new-url]

Problem 2: Permission denied (publickey)

  • Verify SSH key is added to Bitbucket account
  • Check ssh-agent is running: eval "$(ssh-agent -s)"
  • Add key to agent: ssh-add ~/.ssh/id_ed25519

Problem 3: Merge conflicts in pull request

  • Pull latest changes from target branch
  • Resolve conflicts locally in your editor
  • Commit resolved files and push to update PR

Conclusion

Bitbucket is a robust Git repository hosting platform that combines code management with powerful CI/CD capabilities. Its deep integration with Atlassian tools makes it ideal for teams seeking a unified development and project management experience.
